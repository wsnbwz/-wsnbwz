简单搭建http服务

一、安装httpd

执行命令

yum install httpd

二、配置httpd

vi /etc/httpd/conf/httpd.conf

三、修改配置文件

3.1 修改端口号

# Listen: Allows you to bind Apache to specific IP addresses and/or
# ports, instead of the default. See also the <VirtualHost>
# directive.
#
# Change this to Listen on specific IP addresses as shown below to
# prevent Apache from glomming onto all bound IP addresses.
#
# Listen 12.34.56.78:80
Listen 80 # http监听端口根据实际情况修改 ---如果开启了sellinux和防火墙,修改了默认端口80，需要放行防火墙端口和selinx放行端口

端口放行教程（假如我们自己定义监听端口为8080）

防火墙放行
[root@localhost /]# firewall-cmd  --add-port=8080/tcp --permanent
success
[root@localhost /]# firewall-cmd  --reload
success
[root@localhost /]# firewall-cmd   --list-all
public (active)
target: default
icmp-block-inversion: no
interfaces: ens33
sources:
services: ssh dhcpv6-client
ports: 8080/tcp
protocols:
masquerade: no
forward-ports:
source-ports:
icmp-blocks:
rich rules:
[root@localhost /]#

selinux放行
[root@localhost /]# semanage port -a -t http_port_t -p tcp 8080
ValueError: 已定义端口 tcp/8080
[root@localhost /]# 
[root@localhost /]# systemctl restart httpd
[root@localhost /]#

“注意如果你用的是Centos系统的话而且是最小化安装可能会没有semanage命令，需要自己安装，查找这个命令的包yum provides semanage，包安装yum install -y policycoreutils-python”

3.2 修改目录权限

#
# Deny access to the entirety of your server's filesystem. You must
# explicitly permit access to web content directories in other
# <Directory> blocks below.
#
<Directory />
AllowOverride none
# Require all denied 默认设置，拒绝所有访问请求
Options All # 显示目录结构，以便用户浏览下载
</Directory>

3.3 修改共享目录–如果修改了默认路径需要修改路径的上下文

#
# DocumentRoot: The directory out of which you will serve your
# documents. By default, all requests are taken from this directory, but
# symbolic links and aliases may be used to point to other locations.
#
DocumentRoot "/data/" # 根据实际情况修改
 
#
# Relax access to content within /var/www.
#
<Directory "/data"> # 根据实际情况修改
AllowOverride None
# Allow open access:
# Require all granted
# Allow from all
</Directory>

修改定义网站目录上下文–如果不修改需要关闭selinx不然网站无法访问

[root@localhost ~]# mkdir /data
[root@localhost ~]# ls -ldZ /data/		#查看默认上下文关系-object_r:default_t
drwxr-xr-x. root root unconfined_u:object_r:default_t:s0 /data/

[root@localhost ~]# semanage fcontext -a -t httpd_sys_content_t "/data(/.*)?"
[root@localhost ~]# restorecon -R -v /data
restorecon reset /data context unconfined_u:object_r:default_t:s0->unconfined_u:object_r:httpd_sys_content_t:s0
[root@localhost ~]# ls -ldZ /data/		#查看现在上下文关系-httpd_sys_content_t
drwxr-xr-x. root root unconfined_u:object_r:httpd_sys_content_t:s0 /data/
[root@localhost ~]#

记不住命令的可以man帮助查看命令
[root@localhost ~]# man semanage
[root@localhost ~]# man 8 semanage-fcontext 

可以根据自己需求配置

http全局配置文件，文件选项参数含义–郭老师上课写的笔记内容

配置文件选项
ServerRoot                        # 服务目录
Listen 80                         # 监听的端口； Listen   IP:PORT # 如果不写IP表示监听本机所有的可用IP地址，如果注明IP则表示监听在指定的IP地址之后。Listen选项可以出现多次，表示监听不同IP以及不同端口。
ServerAdmin                       # 管理员邮箱
AllowOverride none                # AllowOverride定义每个目录下.htaccess文件中的指令类型，通常设置为None，什么都不可以放；ALL所有都可以放
Require all denied                # 全部拒绝访问
Require all granted               # 全部允许访问
Require  ip   X.X.X.X             # 允许指定 IP地址访问
Require  host    hostname         # 允许指定的主机名访问
DocumentRoot                      # 存放网页数据的目录
Options Indexes FollowSymLinks    # Options选项，Indexes 允许目录浏览，当客户端指定要访问的目录，但没有指定要访问的文件，且目录下不存在默认文档时，显示该目录中的文件及子目录列表索引。 FollowSymLinks 允许跟踪符号链接文件所指定的源文件，跟踪符号链接，如果目录下存在软连接，则可以直接访问软连接指向的文件。
DirectoryIndex                    # 默认可识别的网页
ErrorLog ";logs/error_log";       # 错误的日志文件
LogLevel warn                     # 日志等级
ErrorDocument                     # 错误重定向

七、配置http网站用户访问站点

我们在touch /etc/httpd/conf.d/man.conf重新编写一个配置文件

用户认证文件选项配置参数解释

想要认证需要在httpd.conf中找到想要认证页面的<Directory/>添加认证语句，
AuthType Basic         # 是设置加密方式，基础
AuthName "SOS Dan"     # 是设置域名称
AuthUserFile /var/jff2014/user.pw   # 是密码保存路径
Require user jiaofangfang           # 是指明用户名

编写配置文件

[root@localhost data]# cat /etc/httpd/conf.d/man.conf 
Listen 8080
<VirtualHost *:8080>
    DocumentRoot "/data"
    Directoryindex index.html
        <Directory "/data">
            AllowOverride None
            AuthType Basic
            AuthName "SOS Dan"
            AuthUserFile /wsnb/jnds/user.pw
            Require user jnds
        </Directory>
</VirtualHost>
[root@localhost data]#

创建用户，创建用户密码信息路径，添加http用户和密码

为了安全我们创建保存用户密码文件的时候，为了安全考虑可以在前面加上“.”一点，吧文件弄成隐藏文件,例如吧下面的user.pw—->.user

[root@localhost data]# mkdir /wsnb/jnds/
[root@localhost data]# touch /wsnb/jnds/user.pw
[root@localhost data]# htpasswd -c /wsnb/jnds/user.pw jnds
New password: 
Re-type new password: 
Adding password for user jnds
[root@localhost data]#

重启httpd服务

如果没有放行selinux和防火墙的自行去放行上面有教程

还有网页目录的上下文

[root@localhost ~]# systemctl restart httpd
